## 前置准备

### 环境准备

1. **安装最新 APK：**
   ```powershell
   adb -s emulator-5554 install -r app\build\outputs\apk\release\app-release.apk
   ```

2. **确保有足够历史的测试会话：**
   - 至少一个包含多种工具调用（edit, write, bash, glob/grep, task）的会话
   - 至少一个长对话（20+ 条消息）用于测试加载更多
   - 至少一个包含 Markdown 表格的对话

3. **清除 App 数据（可选，用于干净测试）：**
   ```powershell
   adb -s emulator-5554 shell pm clear dev.minios.ocremote
   ```

### 测试数据要求

| 需要的会话类型 | 获取方式 |
|--------------|---------|
| 包含工具调用的会话 | 让 Agent 执行 `edit file`, `bash command`, `grep pattern` |
| 长对话 | 连续发送 20+ 条消息 |
| 包含表格的对话 | 让 Agent 生成 Markdown 表格 |
| 包含行内代码的对话 | 让 Agent 回复包含 `inline code` 的内容 |

## 详细测试步骤

### T1.1: 流式输出时手动上滑不回弹 [P0]

**目的：** 验证 Task 1 — 去掉 pendingCount > 0 后，流式输出中上滑不被拉回

**步骤：**
1. 打开一个会话
2. 发送一条需要 Agent 长时间回复的消息（如 "写一个 500 字的故事"）
3. **在 Agent 开始流式输出后**，立即用手指向上滑动（查看更早的内容）
4. 观察 3 秒

**预期结果：**
- ✅ 页面停留在用户手动滑到的位置
- ✅ Agent 继续流式输出（状态栏/输入框显示 busy）
- ✅ 不被强制拉回底部

**失败标志：**
- ❌ 页面自动滚回底部
- ❌ 每次新 token 输出都把用户拉回

---

### T1.2: 自动滚到底部 FAB 正常工作 [P0]

**目的：** 验证 autoScrollEnabled 逻辑仍然工作

**步骤：**
1. 接 T1.1 的状态（用户已上滑，不在底部）
2. 观察是否出现"滚动到底部" FAB 按钮
3. 点击 FAB

**预期结果：**
- ✅ 上滑后出现"滚动到底部" FAB
- ✅ 点击后平滑滚动到最新消息
- ✅ 之后 Agent 继续输出时自动跟随（因为回到了底部）

---

### T1.3: Agent 请求权限时不强制滚到底 [P0]

**目的：** 验证 pendingCount > 0 被正确移除

**步骤：**
1. 打开一个会话
2. 发送需要 Agent 请求确认的操作（如删除文件）
3. 在 Agent 处理过程中，手动向上滑动
4. 等待 Agent 发出权限/问题请求

**预期结果：**
- ✅ 权限请求出现在列表底部
- ✅ 用户当前位置不被强制拉到底部
- ✅ 手动滑到底部后能看到权限请求卡片

**替代方案（如果难以触发权限请求）：**
- 发送 `/compact` 等斜杠命令后上滑，观察是否被强制拉回

---

### T2.1: Edit/Write 卡片 Pending 状态可点击 [P0]

**目的：** 验证 EditToolCard 和 WriteToolCard 无条件可点击

**步骤：**
1. 打开包含 edit/write 工具调用的历史会话
2. 找到处于 Pending 或 Running 状态的工具卡片（或刚发送需要 edit 的命令）
3. 点击卡片头部区域

**预期结果：**
- ✅ 卡片响应点击（触觉反馈）
- ✅ 展开箭头图标旋转
- ✅ 展开区域显示（Pending 状态时内容为空，但展开区域存在）
- ✅ 再次点击可收缩

---

### T2.2: Bash/Search 卡片无输出时可点击 [P0]

**目的：** 验证 BashToolCard 和 SearchToolCard 无条件可点击

**步骤：**
1. 打开包含 bash/grep 工具调用的历史会话
2. 找到无输出的 bash 卡片或 search 卡片
3. 点击卡片头部

**预期结果：**
- ✅ 点击有响应（触觉反馈）
- ✅ 可展开/收缩
- ✅ 展开后内容区为空但布局正常

---

### T2.3: Task 卡片子会话导航正常 [P0]

**目的：** 验证 TaskToolCard 保留 subSessionId 优先导航

**步骤：**
1. 打开包含 Task 工具调用的会话（Agent 使用了 subagent）
2. 点击有子会话 ID 的 Task 卡片

**预期结果：**
- ✅ 点击后导航到子会话界面（不是展开/收缩）
- ✅ 子会话界面显示子会话内容
- ✅ 返回后原会话正常

3. 找到没有子会话 ID 的 Task 卡片（或普通工具结果）
4. 点击

**预期结果：**
- ✅ 展开/收缩（不导航）

---

### T2.4: 展开箭头始终可见 [P0]

**目的：** 验证所有工具卡片展开箭头图标始终显示

**步骤：**
1. 打开包含各种工具调用的会话
2. 逐一检查 Edit, Write, Bash, Read, Search, Task, TodoList, PatchCard
3. 确认每个卡片都有展开箭头

**预期结果：**
- ✅ 所有工具卡片头部右侧有展开箭头图标
- ✅ Pending 状态的卡片也有箭头
- ✅ 无输出的卡片也有箭头

**目的：** 验证 scrollBy 补偿机制工作

**步骤：**

4. 观察上方内容是否移动

**预期结果：**
- ✅ 点击展开时，上方消息保持在原位
- ✅ 工具卡片内容向下延展
- ✅ 无视觉跳动/闪烁

**失败标志：**
- ❌ 上方内容整体上移
- ❌ 视觉跳动

### T4.1: 加载更多后内容不闪烁 [P1]

**目的：** 验证 mergeMessages 保留 SSE parts

**步骤：**
1. 打开一个有长对话历史的会话（20+ 条消息）
2. 滑到顶部
3. 点击"加载更多"
4. 观察 3 秒

**预期结果：**
- ✅ 已有内容不闪烁/消失
- ✅ 新加载的内容正常追加到上方
- ✅ 加载指示器消失

**失败标志：**
- ❌ 已显示的内容短暂消失然后重新出现
- ❌ Agent 回复的内容变空后重新填充

---

- ✅ 同一次 Agent 回复的多个 Part 仍然合并在一个气泡中
- ✅ 不出现被截断的单独 Part 气泡
